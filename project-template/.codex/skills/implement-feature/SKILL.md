# Implement Feature

## Workflow

1. Restate the requested behavior and success criteria.
2. Read the relevant files before making changes.
3. Implement the smallest end-to-end change that satisfies the request.
4. Add or update tests for changed behavior.
5. Run focused verification and report what was checked.

## Rules

- Keep the change tightly scoped to the request.
- Reuse existing project patterns before adding new abstractions.
- If a requirement is ambiguous, state the assumption before proceeding.
