# Code Review

## Workflow

1. Inspect the diff or changed files first.
2. Focus on correctness, regressions, security, and test coverage.
3. Prefer concrete findings over style preferences.
4. Reference files and explain user impact.
5. State residual risk if verification is incomplete.

## Output Standard

- Present findings first, ordered by severity.
- Keep summaries brief.
- If no issues are found, say so explicitly.
